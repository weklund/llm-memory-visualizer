## Learning goal

Explain **internal** and **external** fragmentation of KV memory, and how **paged** allocation (PagedAttention-style) maps logical tokens to physical pages.

## Builds on lessons 3–4

Caches are large; many requests share one GPU. Naïve “one huge contiguous slab per sequence” wastes capacity.

## The contiguous reservation problem

If a system pre-reserves a max-length contiguous region per request:

- **External fragmentation:** free memory exists in holes too small to place a new reservation.
- **Internal fragmentation:** the last partially filled region wastes space up to almost a full block/page.

Paged designs borrow the OS idea: fixed-size **pages** (blocks of `P` tokens) + a **block table** from logical indices → physical page IDs (`S05`).

## Formula anchors

<Formula>
{`logical_blocks = ceil(S / P)
internal_waste_tokens ≤ P − 1   (last partial page)

Physical pages need not be contiguous.
Shared pages (prefix reuse) appear in lesson 6.`}

</Formula>

## Mental model (PagedAttention)

1. Split each sequence’s KV into blocks of `P` tokens.
2. Allocate physical blocks on demand.
3. Attention kernels gather K/V via the block table (non-contiguous OK).
4. Optional copy-on-write sharing enables prefix reuse (lesson 6).

Paper-reported waste reductions are **workload-specific**—quote them as paper results, not universal constants (`docs/sources.md` validation list).
